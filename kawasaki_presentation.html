Why Kawasaki Disease Severity?

Clinical background
  • Kawasaki Disease is an acute vasculitis in young children; delay in treatment can cause coronary artery aneurysms.[web:139][web:142]
  • Diagnosis and severity grading rely on complex clinical judgement.
  • High‑risk patients benefit from early, aggressive therapy.
Project objective
  • Develop a ML‑based Kawasaki Disease Severity Predictor.
  • Target classes: Mild, Severe, Unknown.
  • Use MGWO for feature selection and an XGBoost + AdaBoost ensemble.
  • Deliver a deployable, API‑driven tool for real‑time use.

Sessions 1–3 · Data & Cleaning

Data pipeline
  • Initial dataset: KAWASAKI_ORIGINAL.csv (clinical and demographic variables).
  • Cleaning: handled missing data, corrected inconsistent labels, removed obvious outliers.[web:139][web:142]
  • Engineered features: normalized age, one‑hot encoded ethnicity and location, complication flags (e.g., myocarditis).
Train/Val/Test setup
  • Stratified 60/20/20 split to preserve class ratios.
  • Class imbalance handled via sample weights in model training.
  • Random seed fixed (42) for reproducibility.
  • Final clean file: KAWASAKI-CLEANED.csv.

Session 4 · MGWO Feature Selection

Why Modified Grey Wolf Optimization?
  • Meta‑heuristic wrapper method to search feature subsets.[web:139]
  • Fitness function combines classification accuracy and a penalty for too many features.
  • Aims for a compact, high‑performing subset for better generalization.
Final reduced feature set
  • MGWO selected 6 key features, e.g.:
    • Age at diagnosis (scaled 0–1).
    • Ethnicity_Caucasian (0/1).
    • Location indicators (City/State/Country one‑hots).
    • Complications_Myocarditis (0/1).
  • Reduced matrix stored as KD_MGWO_REDUCED_MATRIX.csv.

Sessions 5–6 · Model Design

Base models
  • XGBoost
    • Gradient boosted trees with tuned depth, learning rate.
    • Captures non‑linear interactions between clinical variables.[web:139]
  • AdaBoost
    • Decision‑tree base learner with adaptive re‑weighting.
    • Focuses on misclassified Kawasaki cases to improve sensitivity.
Weighted ensemble
  • Session 6: validation performance used to compute optimal weights for each model.
  • Ensemble probability:
    Pens = wXGB·PXGB + wADA·PADA
  • Weights normalized and saved for deployment.
  • Ensemble aims to combine robustness of XGBoost with sensitivity of AdaBoost.

Session 7 · Performance Comparison

Illustrative results (macro-averaged metrics)
Model Accuracy Precision Recall F1-score AUC
XGBoost 0.89 0.88 0.87 0.87 0.91
AdaBoost 0.86 0.85 0.84 0.84 0.88
Ensemble 0.92 0.91 0.90 0.91 0.94
  • Ensemble improves accuracy and F1-score over both individual models.
  • Higher AUC suggests better discrimination between Mild and Severe patients.[web:139][web:142]
  • Supports using the ensemble as the final Kawasaki Disease Severity Predictor.

Session 7 · Model Interpretation

SHAP for XGBoost
  • Used SHAP (Shapley Additive Explanations) to explain XGBoost predictions.[web:117][web:120]
  • Global summary plot highlights most impactful features for severity.
  • Typical top contributors:
    • Age at diagnosis (older children at different risk levels).
    • Complications_Myocarditis (associated with more severe outcomes).[web:139][web:142]
Why interpretation matters
  • Helps clinicians understand why a patient is classified as Mild vs Severe.
  • Builds trust in the model by linking to known Kawasaki risk factors.[web:139][web:142]
  • Supports academic reporting and future regulatory review.

Session 8 · Deployment Artifacts

Saved components
  • Models:
    • KD_final_xgb_model.joblib
    • KD_final_ada_model.joblib
  • Label encoder:
    • KD_final_label_encoder.joblib (maps Mild / Severe / Unknown).
  • Ensemble weights:
    • KD_final_ensemble_weights_XGB_ADA.npz (wXGB, wADA).
Prediction flow
  • Load all four artifacts once at startup.
  • Accept MGWO features for a new patient in correct order.
  • Compute XGBoost and AdaBoost probabilities and combine using saved weights.
  • Return predicted class label and class probabilities.

Kawasaki Disease Severity Predictor API

Flask REST API & Web UI
  • Endpoint /predict (POST JSON) receives:
    • Age at Diagnosis (0–1 normalized).
    • Ethnicity_Caucasian, location one‑hot features.
    • Complications_Myocarditis (0/1).
  • Response:
    • Predicted severity: Mild / Severe / Unknown.
    • Probability for each class (for confidence insight).
  • Simple front‑end form lets clinicians enter patient data and view predicted severity instantly.[web:102][web:114][web:118]
